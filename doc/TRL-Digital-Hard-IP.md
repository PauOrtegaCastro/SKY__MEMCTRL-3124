# Intro

## TRL 1 - Basic principles observed

- [ ] Is the IP functionality clearly described?
- [ ] Have the potential applications of the IP been identified and described?

## TRL 2 - Concept formulation

- [ ] Is the IP architecture described at the block level?
- [ ] Are the IP requirements and specifications clearly described, including I/O descriptions and whether it includes a standard interface?

## TRL3 - Experimental proof of concept

- [ ] Have the core functionalities of the system been implemented at RTL level?
- [ ] Has the RTL model been provided?
- [ ] Has a complete behavioral simulation flow been provided?
- [ ] Has a testbench been provided?
- [ ] Has the basic functionality of the IP been demonstrated?

## TRL4 - Component prototype

- [ ] Has the RTL been fully developed with all features implemented?
- [ ] Has the design been synthesized (e.g. FPGA or ASIC)?
- [ ] Has a complete post-synthesis simulation been carried out (e.g. FPGA or ASIC) or a logic equivalence check (LEC) report provided?
- [ ] Have preliminary post-synthesis Power, Performance, and Area (PPA) results been provided (e.g. on FPGA or ASIC)?
- [ ] Has the IP been able to run preliminary application code (e.g. an accelerator running machine learning code on simulation or FPGA)?

## TRL5 - Subsystem designed and tested

- [ ] Has full verification been completed and documented, including 100% code coverage with waivers?
- [ ] Has the Place and Route stage been successfully completed and corresponding constraints provided?
- [ ] Has the LEC between the RTL and the post P&R design been performed?
- [ ] Have final post-layout Power, Performance, and Area (PPA) results been provided?
- [ ] Have the post–place and Route simulations been performed, and are the corresponding testbenches provided?
- [ ] Are there clear instructions on how to use and integrate the IP?

## TRL6 - Functional (Alpha) prototype

- [ ] Have all IP views beeen completed?
- [ ] Has full functional verification been completed with documented coverage?
- [ ] Has signoff been completed in the target technology node?
- [ ] Has the IP been characterized across voltage, temperature, and process corners?

## TRL7 - Field demonstration prototype

- [ ] Has the silicon prototype (e.g. through an MPW) been demonstrated on an application development board or another platform, and tested in a relevant environment?
- [ ] Has the application development board been provided and fully documented?
- [ ] Have the key performance metrics at the application level been provided?

## TRL8 - Beta prototype (commercial ready system)

- [ ] Has the IP undergone full qualification, including design-for-manufacturing and design-for-testability features such as scan chains, BISTS and related structures?
- [ ] Has the IP passed industry-standard compliance and certification testing?
- [ ] Has this IP been fabricated in full mask production?
- [ ] Has the IP been deployed in an end-use prototype?
- [ ] Has the IP been tested in a pilot production run or customer field trial?

## TRL9 - Commercial application

- [ ] Has this IP been used in product production?
- [ ] Has this IP been comercially deployed?
- [ ] Has the product reliability been demonstrated in the field?
- [ ] Are the IP authors providing support for bug fixing or enhancement requests?
- [ ] Does the IP documentation include complete supporting material, including training resources?
